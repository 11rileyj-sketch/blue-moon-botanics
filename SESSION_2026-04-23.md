## Bug fix: infinite rerun loop

**Root cause:** `st.logout()` triggers a full-script rerun every time it renders as a persistent widget in the sidebar. Confirmed via stack trace debugging and bisection (commented out full sidebar → stable; re-enabled sidebar without `st.logout()` → stable).

**Secondary cause (already fixed prior to this session):** `st.session_state` writes inside a `@st.cache_data` function (`fetch_beta_users`) — removed in commit `4fc25df`.

**Fix:** `st.logout()` is no longer rendered persistently. A plain `st.button("Sign out")` in the sidebar sets `st.session_state["_sign_out"] = True` and calls `st.rerun()`. At the top of the script (right after the auth gate), `if st.session_state.get("_sign_out"): st.logout(); st.stop()` handles the actual logout on that one rerun.

**Attempts that didn't work:**
- Manual Auth0 logout URL (`/v2/logout`) — clears Auth0 session but not Streamlit's session cookie
- `@st.fragment` wrapper around `st.logout()` — fragment scope doesn't contain auth-triggered reruns

## `species_enrichment.py` — Vertex AI + two-pass architecture

**Auth backend:** switched from AI Studio (`api_key=GEMINI_API_KEY`) to Vertex AI (`vertexai=True, project='gen-lang-client-0299334879', location='us-central1'`). Requires `GOOGLE_APPLICATION_CREDENTIALS` env var pointing at `bmb-enrichment-key.json`.

**Two-pass loop:**
- Pass A (scientific) — calls `SCIENTIFIC_PROMPT`, writes JSON blob to `Enrichment JSON` on Species Library. Must succeed before Pass B runs.
- Pass B (cultural) — calls `CULTURAL_PROMPT`, writes each mention to Cultural Mentions table with `Status: Pending`. Non-blocking — scientific blob is already saved if cultural fails.

**Retry logic:** 60s backoff on 429/503 before trying next model in `CANDIDATE_MODELS`.

**Cultural pass prerequisite:** Cultural Mentions table does not yet exist in Airtable. Scientific pass will run fine; cultural pass will fail gracefully until table is created.
